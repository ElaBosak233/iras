# AI 赋能的智能简历分析系统

## 背景

在招聘流程中，快速筛选和分析大量简历是一项耗时的工作。你的任务是构建一个后端服务，能够自动解析上传的简历（PDF 格式），提取关键信息，并利用 AI 模型对简历进行评分和关键词匹配，帮助招聘者快速筛选候选人。

### 技术要求
- 后端开发语言：Python（使用 uv 作为包管理器）
- API 规范：RESTful
- LLM 框架：Langchain / Langgraph
- LLM API：[硅基流动](https://docs.siliconflow.cn/cn/api-reference/chat-completions/chat-completions)

## 功能模块

### 模块一：简历上传与解析 必选

- 提供接口，支持上传 单个 PDF 格式 的简历文件
- 解析 PDF 内容，提取文本信息（需兼容多页简历）【这个可以使用 DeepSeek OCR 模型】
- 对提取文本进行清洗和结构化处理（去除冗余字符、合理分段等）

### 模块二：关键信息提取 必选
利用 AI 模型从简历文本中提取以下关键信息：

类别|字段|是否必须
:-:|:-:|:-:
基本信息|姓名、电话、邮箱、地址|✅ 必选
求职信息|求职意向、期望薪资|✅ 必选
背景信息|工作年限、学历背景、项目经历|✅ 必选

### 模块三：简历评分与匹配 必选
- 提供接口，接收招聘岗位的需求描述（文本）
- 对岗位需求进行关键词提取和分析
- 将解析后的简历信息与岗位需求进行匹配，计算匹配度评分（技能匹配率、工作经验相关性等）
- 利用 AI 模型对匹配度进行更精准的评分【这个可以使用 GLM 5.1】

### 模块四：结果返回与缓存 必选
- 以 JSON 格式 结构化返回解析结果、关键信息和匹配度评分
- 加分项：实现缓存机制（如 Redis），对已解析和评分的简历进行缓存，避免重复计算

### 模块五：前端页面 必选
- 使用 Vite + React + TypeScript + Tailwind CSS + Shadcn/ui 完成一个简洁可用的交互页面
- 使用 Biome 作为 Linter 和 Formatter
- 使用 `ncu` 来尽可能保证包是最新的
